# Configuration & Logging Module (`config.py`)

## Overview & Purpose
The `config.py` module defines centralized configuration parameters and standardized logging utilities across the entire backend application. It ensures that system paths, database locations, vector dimensions, DiskANN tuning parameters, and log formatting are uniformly managed from a single location.

---

## Classes & Public APIs

### `class Config`
A static namespace container that defines defaults and global settings used by the data layer, ingestion pipeline, vector database, and conversational memory modules.

#### Attributes & Parameters
| Attribute | Type | Default Value | Description |
| :--- | :--- | :--- | :--- |
| `APP_NAME` | `str` | `"Final Year Project Backend"` | Human-readable application title. |
| `DATASET_PATH` | `Path` | `Path("dataset").resolve().parent / "dataset"` | Absolute path to the raw dataset directory. |
| `DEBUG` | `bool` | `False` | Controls diagnostic execution modes and logging verbosity (`DEBUG` vs `INFO`). |
| `LOG_FILE` | `str` | `"log/app.log"` | Relative or absolute path where application log files are written. |
| `ABS_PATH` | `str` | *Computed parent path* | Absolute path to the root application directory (`backend/app`). |
| `DATA_DIR` | `str` | `os.path.join(ABS_PATH, "data")` | Directory where intermediate data files are persisted. |
| `DB_PATH` | `str` | `"hierarchical_db"` | SQLite database filename/path used by hierarchical chunking storage. |
| `EMBEDDING_MODEL` | `str` | `"sentence-transformers/all-MiniLM-L6-v2"` | HuggingFace model identifier for generating dense vector embeddings. |
| `MODEL_PATH` | `Optional[str]` | `None` | Path to a local fine-tuned embedding model checkpoint if used. |
| `INDEX_PATH` | `str` | `"./disk_ann_index"` | Directory where DiskANN dynamic memory vector index files are saved. |
| `DISTANCE_METRIC` | `str` | `"l2"` | Vector similarity metric used by DiskANN (`"l2"`, `"cosine"`, or `"inner_product"`). |
| `VECTOR_DTYPE` | `type` | `np.float32` | NumPy data type for vector elements stored in indexes and databases. |
| `DIMENSIONS` | `int` | `128` | Dimensionality of dense vector embeddings generated by the embedding model. |
| `MAX_VECTORS` | `int` | `1,000,000` | Maximum capacity limit for the DiskANN index. |
| `COMPLEXITY` | `int` | `100` | Search beam width/complexity parameter during index construction and lookups (`L` parameter). |
| `GRPAH_DEGREE` | `int` | `120` | Graph out-degree parameter (`R` parameter) for the DiskANN Vamana graph. |
| `NUM_THREADS` | `int` | `4` | Worker threads allocated for parallel graph indexing and search queries. |
| `K_NEIGHBORS` | `int` | `9` | Default number of nearest neighbor results (`k`) returned during similarity search. |
| `CONVERSATION_SNAPSHOT_DB` | `str` | `"memory/topic_pool/project_pool/conversation_pool/conversation_snapshot.db"` | Path to the SQLite database storing conversation snapshot metadata and vector ID mappings. |
| `SUMMARY_PATH` | `str` | `"memory/topic_pool/project_pool/conversation_pool/conversation_vectors/summary_vectors/"` | Base directory for storing memory-mapped binary summary vectors. |
| `CUMMULATIVE_VECTOR_PATH` | `str` | `"memory/topic_pool/project_pool/conversation_pool/conversation_vectors/cummulative_vector/"` | Base directory for storing memory-mapped binary cumulative summary vectors. |

### `get_logger(name: str) -> logging.Logger`
Retrieves or initializes a standard Python `Logger` object equipped with both file and console handlers.

#### Parameters
| Parameter | Type | Description |
| :--- | :--- | :--- |
| `name` | `str` | The module name identifying the logger (typically passed as `__name__`). |

#### Return Value
- **Type:** `logging.Logger`
- **Description:** A configured logger instance that formats outputs using timestamp, log level, module name, line number, and message text (`%(asctime)s | %(levelname)-8s | %(name)s:%(lineno)d | %(message)s`).

#### How It Works
1. Checks whether handlers have already been attached to the requested logger instance (`if not logger.handlers:`).
2. Computes the absolute path to `Config.LOG_FILE` and creates the parent log directory (`os.makedirs(..., exist_ok=True)`) if it does not already exist.
3. Attaches a `FileHandler` configured with UTF-8 encoding that logs at `DEBUG` level if `Config.DEBUG` is `True` (otherwise `INFO`).
4. Attaches a `StreamHandler` targeting stdout/console output operating at `INFO` level.
